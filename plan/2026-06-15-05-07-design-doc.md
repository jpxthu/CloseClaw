## 思路

### Gap 1 — tools/disallowedTools 运行时过滤
设计文档要求 `tools` 为白名单、`disallowedTools` 为黑名单，控制 agent 可见工具集。
当前 `ToolRegistry.list_descriptors()` 忽略 `ToolContext` 参数，返回全部注册工具。
`gateway/session_manager/spawn.rs:136` 有注释明确承认 "Per-agent tool filtering is not yet supported"。

实现：
- 在 `PromptGenerationContext` 中新增 `tools: Option<Vec<String>>` 和 `disallowed_tools: Option<Vec<String>>` 字段
- 在 `build_from_workspace` / prompt 构建流程中，将 agent config 的 `tools`/`disallowedTools` 传入 `PromptGenerationContext`
- 在 `list_descriptors` 或其调用方中，根据 `PromptGenerationContext` 过滤工具列表
- 过滤逻辑：`tools` 非 `["*"]` 时仅保留白名单中的工具；`disallowedTools` 非空时排除黑名单中的工具

### Gap 2 — skills 白名单运行时过滤
设计文档要求 `skills` 字段控制 agent 可用 skill 列表。
当前 `skills` 字段存储在 `AgentConfig` 中但 session 创建时未读取过滤。

实现：
- 在 session 创建流程（`spawn.rs`）中，读取 agent config 的 `skills` 字段
- 将 skills 白名单传递给 skill registry 或 prompt 构建流程
- 过滤逻辑：`skills` 非 `["*"]` 时仅保留白名单中的 skill

### Gap 3 — spawn 控制器回退默认值
设计文档定义 `maxSpawnDepth` 默认 `1`、`maxChildren` 默认 `5`。
代码中 `SpawnController::validate()` 无父配置时回退使用 `max_spawn_depth = 3`、`max_children = 16`。

实现：
- 修改 `src/agent/spawn.rs` 中回退默认值：`3u32` → `1u32`，`16u32` → `5u32`
- 更新相关测试用例

### Gap 5 — 用户级 agent 目录路径
设计文档定义用户级路径为 `~/.closeclaw/agents/<id>/`。
代码 `agent_loader.rs:129` 使用 `config_dir.parent().join("agents")`，实际路径为 `~/agents/<id>/`。

实现：
- 修改 `src/config/agent_loader.rs`：`config_dir.parent().unwrap_or(&self.config_dir).join("agents")` → `config_dir.join("agents")`
- 更新相关测试（`manager_reload_tests.rs:61` 等）
- 验证 `daemon/config_reload.rs:44` 中的类似路径是否也需要修改
