# 工具系统 需求

## 概述

Agent 需要通过工具与外界交互——读取文件、修改代码、执行命令、调用外部服务。工具系统为 Agent 提供统一的能力发现、描述注入和安全执行通道，确保 Agent 知道有哪些工具可用、理解如何正确使用、并且操作在安全边界内执行。

## 功能需求

### F1. 工具注册与发现

Agent 需要在对话开始时就了解自己可用的工具，并在必要时按需获取更多细节。

- Agent 在 system prompt 中看到一组工具清单，包含每个工具的名称、分组归属和功能描述
- 高频常用工具的完整说明直接呈现给 Agent，无需额外操作即可直接调用
- 低频工具至少展示名称和风险标记，Agent 可按需查询完整说明
- Agent 可以通过关键词或工具名查找工具——用自然语言描述想做的事情，系统匹配最相关的工具
- 工具清单总长度有上限，超出时尾截断并提示 Agent 如何了解未展示的工具
- 不同模块（IM adapter、会话管理、技能系统等）可以各自提供工具注册到统一注册中心，无需修改核心工具代码

> **交叉引用**：当前 Agent 可用的工具范围（白名单/黑名单），由 [agent §F3](agent.md)（Agent 能力组合）定义。

### F2. 文件读取

Agent 需要读取文件内容来理解代码、配置和数据。

- Agent 可以读取指定路径的文本文件
- Agent 可指定读取范围（起止行号），分段消费文件内容。大文件有默认读取上限，超出时提供续读指引，Agent 可通过续读实现全篇读取
- 当文件内容超出单次读取上限时，Agent 收到明确的续读指引——告知已读行范围、剩余行数和下一步从哪行继续
- Agent 在同一轮对话中重复读取相同文件且文件未变化时，系统告知文件未变更而非重新返回内容
- Agent 可以列出目录结构，浏览文件组织和内容概览
- Agent 可以在文件中搜索文本模式，快速定位关键内容

### F3. 文件写入与编辑

Agent 需要创建新文件、覆盖已有文件，以及对已有文件做精确的文本修改。

> **交叉引用**：当前 Agent 可用的工具范围（白名单/黑名单），由 [agent §F3](agent.md)（Agent 能力组合）定义。

- Agent 可以创建新文件或覆盖已有文件的全部内容，文件不存在时自动创建父目录
- Agent 可以对已有文件做精确的文本替换——指定要被替换的原文片段和替换后的新文本
- 一次编辑可以包含多个替换操作，所有替换基于文件原始内容做匹配，互不干扰
- 替换文本默认在文件中精确匹配到一处。精确匹配失败时尝试模糊匹配（Unicode 规范化、引号归一化、行尾空白处理），降低因格式差异导致的匹配失败。匹配零处或多处时操作被拒绝，除非 Agent 明确要求全文替换
- 同一文件的多次编辑操作串行执行，避免并发写入导致文件损坏；不同文件的操作互不影响
- Agent 可以连续编辑同一文件的不同位置，无需在每次编辑后重新读取。当 Agent 尝试编辑已被自己刚才修改过的区域时，系统拒绝并要求重新读取
- Agent 可一次发起多个独立文件的读写操作，加速多文件处理
- Agent 可以进行版本控制操作——查看变更状态和提交历史、提交修改、推送和拉取代码

> **交叉引用**：Agent 对文件的读/写权限判定，由 [permission §F3](permission.md)（读写维度）定义。

### F4. Shell 命令执行

Agent 需要执行 shell 命令来完成构建、测试、搜索等需要进程交互的操作。

- Agent 可以执行 shell 命令，在指定的工作目录下运行
- Agent 可以为命令设置超时时间，超时后命令转入后台继续运行，Agent 不会被阻塞等待
- 命令输出超出显示上限时，完整输出持久化到文件，Agent 收到文件路径、原始大小和输出末尾预览，可按需读取完整内容
- 执行时间超过阈值的命令自动转入后台继续运行，Agent 不会被阻塞等待，完成后会收到通知

### F5. 后台任务执行

长时间运行的命令不应阻塞 Agent 的后续工作。

- 系统提示中告知 Agent 哪些场景应使用后台执行（预计运行时间长的命令）
- Agent 可以显式将命令提交为后台任务，无需等待即可继续执行后续操作
- Agent 在下一轮对话开始时看到当前 session 中正在运行的后台任务列表
- Agent 不应主动轮询检查后台任务是否完成——完成后会自动收到通知
- 后台命令完成后，Agent 在下一轮对话中自动收到完成通知（含任务标识、输出位置、成功/失败状态）
- 后台任务运行中被交互式提示（如确认对话框）卡住时，Agent 收到警告通知，建议终止并按需重试
- Session 停止时，该 session 的所有后台任务进程被终止，已产生的输出保留在磁盘

### F6. 并发工具调用

Agent 需要能够同时发起多个独立的工具调用，减少等待时间。

- Agent 可以在同一个消息中发起多个无依赖关系的工具调用，系统并行执行
- 多个文件读取操作可以并行执行
- 不涉及同一文件的文件修改操作可以并行执行
- 读取某文件的同时编辑另一个文件可以并行执行
- 读取和编辑同一文件时，读取先于编辑执行
- 标记为资源消耗型的工具执行时 Agent 应控制并发数量

### F7. 工具使用引导

Agent 需要理解每个工具在什么场景下使用、如何高效使用。

- 工具的使用说明能反映当前上下文——当权限受限时说明受限的操作范围，当工作目录确定时给出路径指引
- 工具说明能提示 Agent 如何组合使用——哪些工具搭配效率更高
- 工具说明能指出常见误用模式——哪些操作看似正确但有问题
- 没有上下文感知描述的工具降级为静态描述，功能不受影响

### F8. 命令安全防护

Shell 命令在执行前需要经过安全检查，防止恶意操作。

- 命令执行前进行结构分析，检测是否有攻击模式（如注入绕过、隐藏命令），一旦发现直接拦截并通知 Owner
- 结构不完整或解析不确定的命令标记为可疑，发起审批请求，由 Owner 决定是否执行
- 疑似攻击和结构不确定的嫌疑命令单独记录日志到 Agent workspace 之外的专用目录，日志包含触发消息 ID 和完整 session log 文件引用，供安全调查
- 结构清晰且无风险的命令正常进入白名单匹配流程
- 危险文件路径（系统关键文件、版本控制内部文件等）的操作不进入白名单，必须经 Owner 审批
- 审批请求推送给 Owner 时，附带 Agent 提供的命令用途说明，帮助 Owner 理解命令意图
- 对于非零退出码不代表错误的命令（如 grep 未命中、diff 发现差异），Agent 看到的执行结果中附带退出码的语义说明，避免误判为命令失败
- Agent 可以查询当前的权限状态，了解自己能执行哪些操作

> **交叉引用**：命令级权限审批的入队、推送和回调机制，由 [permission §F4](permission.md)（审批工作流）定义。

### F9. 工具扩展接入

系统的不同模块需要能够向工具注册中心添加自己的工具。

- Session 模块、技能系统、IM adapter 等模块可以将自己的工具注册到统一注册表，Agent 在 system prompt 中看到它们
- 工具注册遵循统一的接口契约，新模块加入时按优先级加入注册链即可
- 注册阶段的工具名冲突被检测和报错

## 非功能需求

- Agent 感知到的工具调用延迟应尽量低——文件读取、工具发现等高频操作应在亚秒级完成
- 工具清单的总长度不能无限增长——需有 token 预算控制，超出时优雅截断并给出探索指引
- Shell 命令执行的超时控制必须有硬性上限，单个命令不能无限期占用资源
- 后台任务的清理必须与 session 生命周期绑定——session 结束时释放所有关联进程资源
- 命令安全检测不应引入明显的执行前延迟
