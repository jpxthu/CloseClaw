# Skills 需求

## 概述

Skills 模块满足用户通过可复用技能插件扩展 agent 能力的核心诉求——用户创建 SKILL.md 文件放入指定目录后，agent 在下次 session 启动时自动发现并加载该技能，无需修改系统代码。

## 功能需求

### F1. 技能即插即用

User 将 SKILL.md 文件放入技能目录后，agent 在下次 session 启动时自动发现并加载该技能。User 无需修改任何系统代码，也无需手动注册或重启 daemon 进程。

### F2. 技能目录层级

技能文件按作用域分层存放，多个层级的同名技能按固定优先级覆盖：

| 层级 | 作用域 | 优先级 |
|------|--------|--------|
| 项目级 | 仅当前项目 | 最高 |
| Agent 专属 | 仅该 agent | 高 |
| 全局 | 所有 agent | 中 |
| 外部复用 | 由 User 配置决定 | 低 |
| 内置 | 所有 agent（系统默认提供） | 最低 |

User 通过在不同层级放置同名技能来实现覆盖——例如用项目级技能覆盖全局同名技能。外部复用层级允许 User 指定外部目录（如其他工具链的技能目录），直接复用其中的技能。

### F3. 技能配置

每个技能通过 SKILL.md 文件头部的 frontmatter 配置其行为，User 无需理解系统内部机制即可控制技能的各项属性：

- **description**（必填）：技能的简短描述，供 agent 判断何时调用
- **when-to-use**：帮助 agent 判断调用时机的提示
- **paths**：声明文件 glob 匹配模式，在 agent 操作匹配文件时自动激活该技能
- **user-invocable**：控制该技能是否出现在可用技能清单中，以及 User 是否可通过斜杠命令直接调用。默认不出现（不声明则 agent 仅可通过条件激活自动触发调用）
- **effort**：技能的成本估算，供 agent 调度时参考

SKILL.md 正文（frontmatter 之后的指令文本）支持变量替换，User 可在正文中使用 `${SKILL_DIR}` 引用技能所在目录路径、使用 `${SESSION_ID}` 引用当前会话 ID。

> 技能仅提供纯 prompt 指令，不携带任何工具权限。工具权限由 agent 级别的配置统一管理，不由单个技能声明。

### F4. 技能清单

Session 启动时，系统向 agent 注入一份可用技能清单（名称、描述、决策提示、条件激活标记），让 agent 在对话开始时就知道有哪些技能可用。未声明 user-invocable 的技能默认不出现在清单中。

**增量注入**：后续 session turn 中若有新技能被激活（条件激活、热重载等），仅增量注入新增条目，不重发全量清单。增量注入不得通过修改 system prompt 实现——必须在 session 层面追加，保证 system prompt 不变、KV cache 不受影响。

清单按技能来源优先级排序（高优先级在前），同来源内按名称字母序排列。技能列表为空时不注入。

若技能声明了文件条件激活（paths 字段），该技能在清单中标注 ⚡ 标记并展示匹配的文件模式列表（如 `⚡ auto-activates on: **/*.rs`）。

对话压缩时，技能清单作为受保护内容原样保留，压缩仅作用于多轮对话部分，确保 agent 在长会话中始终能读取可用技能清单。

### F5. 热重载

User 在 session 运行期间修改或新增 SKILL.md 文件后，技能变更在下次 session 创建、存档恢复或对话压缩时自动反映到 agent 的可用技能清单中。User 无需手动操作也无需关注具体生效时机。

### F6. 条件激活

声明了 paths 字段的技能不在初始技能清单中。当 agent 操作的文件路径匹配某技能的 paths 模式时，该技能自动激活——在下一个 session turn 以增量方式注入技能清单（不含正文），让 agent 知晓该技能可用。agent 后续可按 F7 正常调用。

条件激活的增量注入不得修改 system prompt，不得破坏 KV cache。

### F7. 技能调用

Agent 在对话中根据技能的 description 和 when-to-use 判断是否调用某个技能。调用时系统加载技能正文并注入对话上下文，agent 按技能指令继续执行。

User 也可通过斜杠命令直接调用声明了 user-invocable 的技能。

### F8. 多 Agent 隔离

多个 agent 各自拥有独立的技能空间。Agent 专属目录下的技能仅对该 agent 可见，不会影响其他 agent。各 agent 可使用哪些技能由 agent 级别的配置决定，不依赖技能文件内声明。

### F9. 错误容错

单个技能文件的错误不影响 session 正常运行：

- 技能目录路径不存在或无法访问时，跳过该来源，记录提示
- 单个 SKILL.md 格式错误或必填字段缺失时，跳过该技能，其他技能正常加载
- 同名冲突时，低优先级版本被跳过，记录提示

### F10. 技能创建工具

User 可通过 agent 对话创建或修改技能文件。agent 使用技能创建工具生成符合规范的 SKILL.md 文件，包含正确的 frontmatter 配置和指令正文。

## 非功能需求

- **加载效率**：技能目录扫描和清单注入不应显著延长 session 启动时间
- **稳定性**：技能加载阶段的任何错误都不应导致 session 启动失败或 daemon 崩溃
- **可观测性**：技能加载失败、同名冲突等异常情况应有明确的日志提示，方便 User 排查问题
- **KV Cache 兼容**：技能清单的初始注入和增量更新均不得修改 system prompt 内容，确保 KV cache 不受影响
