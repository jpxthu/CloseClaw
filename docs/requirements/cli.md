# CLI 需求

## 概述

CLI 模块满足用户通过命令行终端与 CloseClaw 交互的需求——既能进行对话交流，也能直接管理 daemon 和系统配置。

## 功能需求

### F1. 终端对话

Owner 可以在终端中与 Agent 进行文本对话。

- 对话以 `closeclaw chat` 启动，在同一个终端进程中持续交互。此功能依赖 daemon 已运行

- 通过 `--agent-id` 参数指定目标 Agent，不同 Agent 的对话保持独立、互不干扰

- 终端对话支持与其他 IM 渠道（如飞书、Discord）完全一致的斜杠指令体系

- 终端渠道的调用者默认为 Owner，无需额外鉴权

- 使用 `/stop` 结束当前对话

> **交叉引用**：`/stop` 的完整语义见 slash 需求文档；不活跃会话的自动归档与清理见 [session §F6](session.md)。

### F2. 终端输出渲染

终端对话的输出应充分利用终端能力，提供可读性良好的格式化展示。

- **ANSI 渲染策略**：根据终端能力渲染格式化输出。终端能力检测与降级逻辑见 [platform §F3](platform.md)（终端能力自适应）

- **Markdown 转 ANSI**：普通文本中的 markdown 格式标记（标题、粗体、斜体、列表、引用、链接、分割线）渲染为对应的 ANSI 样式

- **代码块高亮**：代码块按语言标注注入语法高亮和行号，覆盖常见编程语言

- **Thinking 折叠**：Agent 的思考过程以折叠块形式展示，有首尾边界标记，不打断对话阅读流

- **工具调用与结果展示**：工具调用展示工具名称和参数，工具结果以截断方式展示

- **媒体占位符**：终端不支持的媒体类型（图片、音频、文件）展示为带文件名的占位符，不尝试输出二进制内容

- **DSL 交互元素降级渲染**：按钮、选择器等交互元素在终端中渲染为纯文本提示行（如 `[Button: 标签 (action: xxx)]`），因为终端不支持交互元素

- **终端宽度适配**：各块输出超过终端可用宽度时截断并标注，不产生溢出

- **流式输出**：LLM 流式输出时，内容逐行增量显示，不等待完整回复

### F3. Daemon 生命周期管理

Owner 可以通过 CLI 命令管理 daemon 进程的启停。

- 使用 `closeclaw run` 启动 daemon

- 使用 `closeclaw stop` 停止 daemon

- daemon 未运行时执行 `closeclaw stop` 不报错，正常退出

- daemon 已运行时执行 `closeclaw run` 应拒绝启动新实例

> **交叉引用**：PID 文件机制见 [platform §F1](platform.md)；daemon 启动与关闭过程的完整需求见 [daemon §F1–F2](daemon.md)。

### F4. 配置管理

Owner 可以通过 CLI 管理 CloseClaw 的配置文件。

- **交互式配置向导**：通过引导式问答完成首次配置，向导完成后自动写入配置文件
  - 选择 LLM Provider 后自动应用推荐协议，用户无需了解协议差异
  - 支持一次选择多个可用模型，满足多模型使用场景
  - 凭据输入时字符不回显，输入后显示确认行让用户确认操作成功
  - 自动获取可用模型列表；获取总体失败时自动回退到内置模型清单并显示提示；网络超时、认证失败、凭据无效等错误允许用户重试
  - Ctrl+C 可安全退出向导，不残留不完整配置文件

- **配置校验**：校验现有配置文件格式的合法性，输出校验结果

> **交叉引用**：配置文件结构、安全写入、重载等需求见 [config §F1–F6](config.md)。

### F5. Agent 管理

Owner 可以通过 CLI 查询和管理 Agent 实例。此功能依赖 daemon 已运行。

> **交叉引用**：Agent 配置档案的创建、查询、列表等完整需求见 [agent §F1](agent.md)。

### F6. Skill 管理

Owner 可以通过 CLI 查询 Skill 列表，或触发 daemon 重新扫描技能目录加载新 Skill。此功能依赖 daemon 已运行。

> **交叉引用**：Skill 即插即用的完整语义见 [skills §F1](skills.md)。

### F7. 权限规则查看

Owner 可以通过 CLI 查看权限规则——包括校验规则语法和列出已有规则。

> **交叉引用**：CLI 提供规则语法校验与列表入口（只读）；权限规则的修改通过斜杠指令完成，详见 [permission §F6](permission.md)。

### F8. 跨平台支持

跨平台一致性见 [platform §F5](platform.md)（系统差异隔离）及非功能需求。

## 非功能需求

- 终端 ANSI 能力检测应覆盖所支持平台的主流终端模拟器，确保用户在不同终端环境下获得一致的格式化体验
- 终端渲染应适应用户的终端窗口尺寸，不产生溢出或布局异常
- 代码块语法高亮（含行号）覆盖常见编程语言，未知语言回退纯文本输出
