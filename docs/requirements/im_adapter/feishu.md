# 飞书 需求

## 概述

飞书是 IM Adapter 模块的飞书平台实现。用户通过飞书私聊或群聊与 Agent 交互，Agent 输出以卡片消息、按钮、代码块等形式在飞书中渲染，Agent 可使用飞书平台特有工具（消息操作、日历、文档等）完成平台操作。

## 功能需求

### F1. 飞书入站消息接收

系统接收飞书 webhook 事件，支持以下事件类型：

- **文本消息**（✅ fixture）：提取纯文本正文
- **富文本消息**（✅ fixture，部分 ⬜）：飞书 API 中称为 "post" 类型消息，展开富文本结构为可读文本。富文本由段落组成，每个段落包含若干文本元素。保留：标题、段落、有序/无序列表（含多级嵌套）✅、文本样式（粗体、删除线、下划线及其组合）✅、@提及（渲染为 `@用户名`）✅、引用块 ✅、内嵌媒体占位符 `[图片]` ✅。链接 ⬜ 和邮箱/电话 ⬜ 无 fixture
- **代码块**（⬜ 无 fixture）：代码块结构的解析行为暂基于设计文档推导，无法验证
- **行内代码/超链接**（⬜ 无 fixture）：行内代码和可点击链接的解析行为暂基于设计文档推导
- **引用/回复消息**（✅ fixture）：提取被引用消息的文本内容（截断至 500 字符），拼接在本消息正文前，以引用格式呈现
- **话题消息**（✅ fixture）：提取话题 ID，用于回复定位。话题 ID 不参与会话路由
- **非文本消息**（⬜ 独立图片/文件/语音消息无 fixture）：独立非文本消息不进行语义理解，记录日志后丢弃
- **富文本内嵌媒体**（✅ fixture）：富文本消息中内嵌的图片等媒体内容以 `[图片]` 等占位符表示
- **表情消息**（✅ fixture）：消息中的表情符号渲染为文本占位符形式（如 `[OK]`、`[赞]`）

入站解析后输出归一化消息，包含平台标识（feishu）、发送者、会话 ID、消息正文。

### F2. 飞书消息事件处理

除消息接收外，系统需处理以下飞书事件（全部 ✅ fixture）：

- **消息回应** `reaction.created`（✅ fixture）：用户对消息添加表情回应时，系统应感知并可用作 Agent 交互信号
- **卡片按钮点击** `card.action.trigger`（✅ fixture）：用户点击卡片中按钮时，系统接收按钮动作并路由到对应处理逻辑
- **机器人加入群聊** `bot.added`（✅ fixture）：机器人被添加到群聊时，系统应知晓并可用于初始化群聊会话

### F3. 飞书消息发送

> ⚠️ 消息发送无 webhook fixture（发消息是主动 API 调用，非接收事件），需用飞书 API 文档验证。

Agent 的回复通过飞书发送给用户。支持两种消息类型：

- **文本消息**：纯文本、无格式标记的单行内容
- **卡片消息**：含格式标记（标题、粗体、斜体、代码块、列表、引用、链接、分割线、换行）或含交互按钮或多内容块的内容

消息格式自动选择，用户无需关心底层发送方式。

### F4. 飞书卡片渲染

Agent 输出的富格式内容需正确渲染为飞书卡片。

**标题提取**：首行 `#` 标题提取为卡片标题（蓝色模板），标题行不进入正文。

**正文元素映射**：

| 内容 | 飞书卡片呈现 | Fixture |
|------|------------|--------|
| `# 标题` | 卡片标题（蓝色） | ⚠️ |
| `##` 及更低级标题 | 原生标题样式 | ⚠️ |
| 粗体 / 斜体 | 原生粗体 / 斜体 | ⚠️ |
| 行内代码 | 行内代码样式 | ⬜ |
| 代码块 | 原生代码块（飞书平台自行语法高亮） | ⬜ |
| 引用 | 引用块 | ✅ |
| 列表（有序/无序） | 原生列表 | ⚠️ |
| 链接 | 可点击链接 | ⬜ |
| 分割线 | 水平分割线 | ⚠️ |

**特殊块渲染**：
- **Thinking 块**：以折叠推理区块的形式展示
- **Tool 调用块**：以工具调用描述形式展示，含工具名和参数概要
- **Tool 结果块**：以工具结果内容形式展示

### F5. 飞书交互按钮

> 按钮渲染 ⚠️ 无独立 fixture；按钮点击事件 ✅ `card-action-trigger` fixture。

Agent 输出中的按钮指令在飞书卡片中渲染为可点击按钮：

- 首个按钮渲染为主要样式（突出显示）
- 后续按钮渲染为默认样式
- 所有按钮平铺在卡片底部操作区域

用户点击按钮后，系统通过 `card.action.trigger` 事件接收按钮动作信息并执行对应操作。

### F6. 飞书平台工具

> ⚠️ 平台工具全部无 webhook fixture（均为主动 API 调用），需用飞书 API 文档验证。

Agent 可使用以下飞书平台工具，在对话中完成平台特有操作：

| 工具分组 | 能力范围 |
|---------|---------|
| 飞书 IM 操作 | 发送消息、撤回消息、编辑消息、表情回应等 |
| 飞书日历 | 日程创建、查询、修改、删除 |
| 飞书任务 | 任务创建、查询、修改、删除 |
| 飞书多维表格 | 表格数据读写、记录管理 |
| 飞书文档 | 文档创建、编辑、读取 |
| 飞书云盘 | 文件上传、下载、管理 |
| 飞书电子表格 | 表格创建、编辑、读取 |

所有工具在首次被 Agent 调用时才初始化（延迟加载）。

### F7. 飞书话题回复

群聊中话题消息的回复需定位到原话题：

- 入站时解析话题 ID（按 `thread_id > root_id > parent_id` 优先级合并）
- 出站时携带话题 ID 定向回复
- 话题 ID 不参与会话路由（会话按群聊粒度管理）

## 非功能需求

### 性能

- 卡片消息渲染不应显著增加消息发送延迟（用户感知与普通 IM 消息相当）

### 可靠性

- 飞书 API 调用失败时系统正常降级，Agent 继续运行不受影响
- 工具调用失败时 Agent 收到清晰错误反馈

### 可观测性

- 非文本入站消息记录日志
- 表情回应和卡片按钮点击记录日志，便于分析交互行为
