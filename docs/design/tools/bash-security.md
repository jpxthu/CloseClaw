# Bash 安全解析

## 概述

Bash 安全解析机制在命令执行前对命令字符串进行结构分析和安全检测，产出可信的解析结果和风险标签。权限引擎依赖解析结果进行精确匹配，攻击检测在权限判断前拦截已知威胁。

CloseClaw 的安全模型不采用 fail-closed（误杀即拒绝），而是**解析失败/不确定时走审批链路，明确攻击时直接拦截并通知 owner**。

## 架构

安全解析由三层协同完成：

```
原始命令字符串
  │
  ├─ 语法解析层 ──→ 产出 AST 语法树
  │     └─ 理解不了的节点类型 → 标记风险
  │
  ├─ 差异检测层 ──→ 检测 parser vs shell 的已知不一致
  │     └─ 匹配攻击模式 → 标记风险 / 直接拦截
  │
  └─ 信任分级层 ──→ 综合判断，产出 trust level + 风险原因
        │
        ├─ trusted（结构清晰无风险）→ 进权限引擎白名单匹配
        ├─ uncertain（解析不确定 / 部分风险）→ 发起审批，附风险说明
        └─ malicious（明确攻击模式）→ 直接拦截，通知 owner
```

### 语法解析层

使用 tree-sitter 解析器对命令字符串做完整语法分析，产出语法树。从语法树中递归提取所有叶节点的简单命令，得到可信的命令数组、参数列表、重定向目标、环境变量。

解析器信任的节点类型为已知的 shell 标准语法结构：命令、管道、列表、重定向、变量赋值、子 shell、heredoc 等。任何不在信任列表中的节点类型均标记为不确定，附解析失败原因。

关键数据产出（每个简单命令）：

| 字段 | 说明 |
|------|------|
| `argv` | 解析后的命令参数数组（引号已展开、变量已替换） |
| `envVars` | 前置的环境变量赋值 |
| `redirects` | 输入/输出重定向（操作符 + 目标） |
| `sourceText` | 原始源码片段（用于权限规则匹配和审批展示） |

### 差异检测层

bash 的 tokenization 行为与 tree-sitter 解析器存在已知的不一致。攻击者可利用这些差异构造"解析器看到 A、bash 执行 B"的命令。差异检测层维护一系列检测器，针对已知的攻击模式。

每个检测器针对一种特定的解析差异。检测到的结果分为两类：

- **攻击模式**：命令明确包含已知注入技巧（如 IFS 注入、`/proc/self/environ` 访问、Zsh 危险内置命令）→ 直接拦截，不进入审批
- **潜在风险**：命令包含可疑特征但无法确定为攻击（如 ANSI-C 引述 `$'...'`、花括号扩展、未引用的重定向）→ 标记为 uncertain，进入审批

检测器按需增加——每发现一种新的绕过技巧，新增一条检测规则。

### 信任分级层

综合语法解析和差异检测的结果，产出三个信任级别：

| 级别 | 判定条件 | 后续行为 |
|------|---------|---------|
| **trusted** | 语法完整、节点全在信任列表、无差异警告 | 产出 `argv[]` → 交给权限引擎做白名单匹配 |
| **uncertain** | 解析失败 / 未知节点 / 部分差异警告 | 附风险原因 → 发起权限审批，等待 owner 确认 |
| **malicious** | 明确匹配攻击模式 | 直接拦截 → 通知 owner，告知攻击类型和命令内容 |

权限引擎收到的命令参数数组来自 trusted 级别的解析结果，匹配更精确——例如 `git commit -m "msg"` 的命令名 + 子命令可以精确匹配为 `git` + `commit` 两级，而非模糊的字符串前缀匹配。

### 退出码语义

命令执行后，安全解析模块根据命令类型解释非零退出码的含义。某些命令的非零退出码不代表错误（如 `grep` 返回 1 表示未命中、`diff` 返回 1 表示文件不同），解释结果附在执行结果中，避免 agent 误判为命令失败。

## 数据流

```
BashTool.call()
  → 安全解析入口
    → 语法解析层：tree-sitter 解析命令字符串
      → 语法完整 + 节点全合规 → 产出 SimpleCommand[]
      → 语法异常 / 未知节点 → 标记 uncertain + reason
    → 差异检测层：遍历检测器
      → 匹配攻击模式 → 标记 malicious + attack type
      → 匹配潜在风险 → 附加到当前信任级别
    → 信任分级层：综合判定
      ├─ trusted → 权限引擎白名单匹配
      │     → 匹配成功 → 执行命令
      │     → 匹配失败 → 发起审批（带 trusted 解析结果供展示）
      ├─ uncertain → 发起审批
      │     → 审批内容：原始命令 + 解析失败原因 + 风险标签
      │     → owner 确认 → 执行命令 / 拒绝
      └─ malicious → 直接拦截
            → 通知 owner：攻击类型 + 命令内容 + 拦截原因
```

## 模块关系

- **上游**：BashTool（命令执行前调用安全解析）
- **下游**：权限引擎（接收 trusted 解析结果做精确匹配）、审批系统（接收 uncertain 结果发起审批）、owner 通知（接收 malicious 拦截通知）
- **无关**：后台任务系统（安全解析在命令入队前完成，不关心执行方式）、IM 适配器（安全解析的结果由权限引擎和通知系统传递，不直接与平台交互）
