# 工具提示词注入

## 概述

工具描述注入机制负责将工具的能力说明注入到 system prompt 中，使 LLM 在对话初始化时理解可用的工具及其用途。

注入分两级：一级索引在 system prompt 的工具区展示所有工具，二级详情在 LLM 主动发现时按需注入。两级配合既保证 LLM 知道有哪些工具，又避免不必要的上下文膨胀。

## 架构

### 注入链路

```
注册中心（所有已注册工具）
  → 索引构建（按分组聚合，生成一级索引）
    → system prompt 构建器（注入工具区）
      → 工具区渲染（格式化输出）
        → LLM 在 system prompt 中看到工具列表
```

### 两级注入

**一级索引**：system prompt 中 `## Tools` 区域。常用工具展示名称、危险度标记和完整详情。延迟工具仅展示名称和危险度标记，不包含详情。按分组聚合输出，每个分组有独立的加载策略标注。

**二级详情**：延迟工具的完整详情。LLM 调用工具发现机制时触发注入，将目标工具的详情追加到当前会话上下文。二级详情的注入时机由工具调用层控制。

### 注入内容

系统提示词中每个工具注入以下信息：

- **工具名**：作为索引键和 LLM 选择的依据
- **危险度标记**：只读工具标 `(read-only)`，破坏性工具标 `(destructive)`。始终显示，不受延迟状态影响
- **行为描述**：工具的完整功能说明（常用工具包含，延迟工具不含）
- **参数格式**：不进入文本提示词，通过 API schema 直接暴露给模型

### 加载策略

工具按加载策略分为两类：

- **常用工具**：初始即注入完整详情，LLM 无需额外步骤即可调用。适用于高频工具（文件操作、元操作、技能发现）
- **延迟工具**：初始仅注入名称和危险度标记。适用于低频或领域专用工具（Git 操作、编程 agent、技能创建）。LLM 需要时通过工具发现机制获取详情

分组头根据组内策略标注：全组延迟标 `(deferred)`，否则标 `(always loaded)`。

### 长度控制

一级索引有总长度上限。超出上限时触发截断，按分组顺序依次输出，超限部分丢弃。分组输出保证原子性——一个分组要么完整输出，要么完全不输出。

## 数据流

### 初始化注入

```
系统启动 → 注册所有工具
  ↓
session 创建 → system prompt 构建
  ↓
调用索引构建 → 遍历注册中心
  ↓
按分组聚合 → 格式化输出
  ├─ 常用工具：**工具名** (危险度): 详情
  └─ 延迟工具：- 工具名 (危险度)
  ↓
长度检查 → 超出上限则截断
  ↓
注入 system prompt 工具区
```

### 按需注入

```
LLM 需要某延迟工具的详情
  ↓
调用工具发现（传入关键词或工具名）
  ↓
注册中心匹配 → 返回目标工具的完整详情
  ↓
将详情追加到当前会话上下文
  ↓
LLM 在后续轮次中使用该工具
```

## 模块关系

- **上游**：system prompt 构建器（调用索引构建，注入工具区）
- **下游**：无（索引构建和注入均在 tools 模块内部完成）
- **无关**：消息处理器（注入发生在 prompt 构建阶段，不经过消息处理链路）、平台渲染器（工具区是纯文本，不涉及平台渲染）
