# 工具关键词索引

## 概述

工具关键词索引为工具发现提供基于自然语言的模糊匹配能力。每个工具在注册时嵌入一组英文关键词，覆盖其核心功能的常见说法，使 LLM 能通过日常用语找到对应工具。

## 架构

### 关键词嵌入格式

关键词以结构化前缀嵌入在工具描述中，格式为 `[keywords: 词1 词2 ...]`。关键词放在描述文本的开头，用方括号包裹，空格分隔。

### 关键词规则

- 每个工具 3 到 10 个关键词，覆盖核心动作和领域
- 只用空格分隔，不含标点
- 关键词来源覆盖：核心动作（创建、查询、发送）、功能领域（消息、任务、日历）、常见说法（日程、提醒）

### 匹配机制

工具发现时对用户输入做关键词匹配，按匹配权重找到最相关的工具。工具名精确匹配权重最高，关键词匹配次之，描述文本匹配辅助。多个关键词匹配时权重累加。匹配后返回得分最高的工具，供 LLM 选择或进一步获取详情。

### 维护原则

新增工具时应同时定义关键词。修改工具功能时同步更新关键词。描述中缺少关键词前缀的建议在质量检查中补充。

## 数据流

```
工具注册时 → 描述中嵌入 [keywords: ...]
  ↓
LLM 使用自然语言描述需求 → 工具发现
  ↓
提取关键词前缀 → 与 LLM 输入匹配
  ↓
返回匹配工具列表，按得分排序
  ↓
LLM 选择目标工具或获取详情
```

## 模块关系

- **上游**：工具发现机制（读取关键词进行匹配）
- **下游**：无（关键词嵌入和匹配均在 tools 模块内部完成）
- **无关**：system prompt 注入（关键词不影响一级索引的输出格式）
