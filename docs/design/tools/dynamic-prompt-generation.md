# 工具提示词动态生成

## 概述

工具提示词动态生成机制使工具的 LLM 使用说明能根据运行时上下文动态调整，而不局限于静态描述文本。

通过此机制，工具描述可以反映当前权限状态、可用工具组合、工作目录信息等动态因素，帮助 LLM 做出更准确的选择。

## 架构

工具描述采用双轨制：Schema 层和 Prompt 层。

```
Tool trait
├── Schema 层 → 给 API 看的调用格式（参数结构，固定不变）
└── Prompt 层 → 给模型看的使用策略（自然语言，随上下文变化）
```

### Schema 层

描述工具的参数结构，以 JSON Schema 格式定义。内容固定不变，作为 API schema 直接暴露给模型，不进入文本提示词。

### Prompt 层

通过动态生成方法输出自然语言使用策略，描述内容包括：

- **何时使用**：什么场景下应该选择这个工具
- **使用原则**：怎么用效率最高（分批、限制范围等）
- **组合建议**：与什么工具搭配使用（取决于当前可用工具列表）
- **权限感知**：当前权限状态下的可用操作范围
- **路径指引**：根据工作目录给出的上下文提示

Prompt 层随调用上下文变化：不同 agent 类型、不同权限级别、不同工作目录下，同一个工具的描述内容会不同。

### 上下文传递

动态生成方法接收运行时上下文，包含：

- **可用工具列表**：当前会话中 LLM 可调用的全部工具
- **权限状态**：当前 agent 的权限级别
- **工作目录**：当前 session 的工作目录
- **agent 定义**：当前 agent 的角色和类型

### 降级策略

如果工具没有 Prompt 层实现，降级为仅输出 Schema 层的静态描述。工具功能不受影响，只是缺少动态使用策略指引。

## 数据流

```
system prompt 构建
  → 遍历已注册工具
    → 对每个工具：
      ├─ 获取 Schema 层（参数结构，给 API 用）
      └─ 调用 Prompt 层生成器（传入当前上下文）
           → 生成自然语言使用策略
              → 合并到工具描述中
    → 注入 system prompt 工具区
```

## 模块关系

- **上游**：system prompt 构建器（在构建工具区时调用 Prompt 层）
- **下游**：工具注册中心（提供工具实例和注册信息）
- **无关**：工具调用层（Prompt 层只影响 LLM 的选择，不影响工具执行）、权限引擎（Prompt 层读取权限状态但不修改）
