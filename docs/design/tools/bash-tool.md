# Bash 工具

## 概述

BashTool 为 agent 提供执行 shell 命令的能力，是 CloseClaw 从文件操作进化到进程执行的核心能力。

通过 BashTool，agent 可以执行构建命令、系统工具调用、外部脚本和命令行工具，与本地开发环境交互。

## 架构

BashTool 的执行链路包含三层保护：

```
传入命令、超时时间和工作目录
  → 权限校验（命令白名单检查）
    → 命令执行（子进程，超时控制）
      → 输出处理（截断、持久化）
```

### 权限校验

命令执行前，通过权限引擎检查命令和参数是否在白名单中。白名单规则按命令和允许的参数列表配置，不在白名单的命令直接拒绝，不执行。

危险路径保护（系统和用户配置文件等敏感路径）由权限模块统一管理：危险路径不出现在白名单规则中，运行时检测到后进入用户审批流程。

### 命令执行

在指定工作目录下以子进程方式执行命令。支持超时控制：默认 60 秒，最长 5 分钟。超时后发送终止信号，结果中标注中断状态。

初始版本不展开路径，命令原样传递给 shell 执行。

### 输出处理

标准输出和标准错误分别捕获。输出超过 30,000 字符时持久化到磁盘文件，返回文件引用而不是完整内容。输出文件超过 64MB 时截断。

命令执行状态通过退出码、中断标记、持久化路径等字段完整返回。

### 安全标记

BashTool 标记为破坏性工具（可修改文件系统）和昂贵工具（可能消耗大量 CPU/IO），在一级索引中显示对应的危险度标记。

## 数据流

```
BashTool 调用（command + timeout + cwd）
  → 权限引擎检查（CommandExec action）
    → 拒绝：返回权限错误
    → 通过：
      → 启动子进程执行命令
        → 等待完成或超时
          → 超时：发送终止信号，标记 interrupted
          → 完成：收集 stdout/stderr
        → 输出长度检查
          → ≤30K 字符：直接返回
          → >30K 字符：持久化到磁盘，返回文件引用
      → 返回执行结果（退出码、输出、中断状态）
```

命令不存在时在标准错误中报告 "command not found"，退出码非零。权限不足时报告 "permission denied"。空命令直接返回错误，不执行。

## 模块关系

- **上游**：agent 运行时（调度工具调用）
- **下游**：权限引擎（命令白名单校验）、操作系统 shell（进程执行）
- **无关**：processor_chain（BashTool 不参与消息出站处理）
