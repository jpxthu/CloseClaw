# 设计文档（docs/design/）写作标准

## 定位

从**产品设计（PM）角度**描述模块架构、职责、内部逻辑流转、与其他模块的信息传递。
讲清楚**最新设计的意图和原因**，不讲历史。

## 写什么

- **模块职责**：这个模块做什么，边界在哪里
- **架构/结构**：模块内部如何组织，子模块划分
- **逻辑流转**：数据在模块内的处理流程
- **模块间信息传递**：与上下游模块的接口/消息格式
- **设计意图**：为什么这样设计，解决了什么问题

## 不写什么

- 历史决策、演进过程
- 代码里直接能看到的字段列表、函数签名
- 详细实现细节
- 未来计划

## 文件命名

`NN-<模块名或主题>.md`，编号与原 design doc 顺序对应（倒序处理）。
