# 代码块渲染

## 概述

代码块渲染是渲染处理器的子功能，负责将消息中的代码块（三反引号包裹的代码段）按语法高亮渲染，替代纯文本展示。

## 架构

代码块渲染集成在各平台的渲染 Processor 中，按平台能力选择不同的高亮策略：

```
ContentBlock[] 中的 Text 块
  ↓
遍历文本，识别 ```lang ... ``` 代码段
  ↓
按平台选择渲染方式：
  ├── 飞书 → 飞书原生 markdown 代码块（平台自行语法高亮）
  ├── CLI  → ANSI 转义序列着色
  └── 其他  → 回退到纯文本代码块
  ↓
渲染输出（含代码高亮）
```

语言标注从 markdown 代码块标记中提取（如 ```rust 表示 Rust 高亮），平台自行决定支持的语言列表。

**平台渲染策略**：

| 平台 | 渲染方式 | 说明 |
|------|---------|------|
| 飞书 | 原生 markdown 代码块 | 飞书消息卡片原生支持 ``` 代码块，平台端自行高亮 |
| CLI | ANSI 转义序列 | 按语言注入颜色码，终端渲染语法高亮 |
| 其他 | 纯文本 | 保留代码块标记，无额外着色 |

## 数据流

```
渲染 Processor 处理 Text 块
  ↓
逐行解析 Text 块内容
  ↓
遇到 ```lang 标记 → 开始代码段
  ├── 记录语言标注
  ├── 收集代码行
  └── 遇到关闭 ``` → 结束代码段
  ↓
按平台选择渲染方式
  ├── 飞书：将整个 ``` 代码块作为 markdown 元素输出
  ├── CLI：逐行注入 ANSI 颜色码后输出
  └── 回退：保留原始 ``` 标记
  ↓
继续后续文本渲染
```

## 模块关系

- **上游**：渲染 Processor（提供 ContentBlock[] 输入）
- **下游**：Gateway（提取平台 payload 传递给 IM Adapter 发送）
- **所属**：各平台渲染 Processor 的内部子功能，不独立为 Processor
