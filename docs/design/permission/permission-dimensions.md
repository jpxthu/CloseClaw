# 权限维度

## 概述

权限维度定义了权限系统管控的八类操作，每类操作独立管理，各自有默认策略和判定规则。

## 架构

八类权限维度各自独立，但存在复用关系：

```
读       ─── 独立维度，限制可读取的文件和目录范围
写       ─── 独立维度，限制可写入的文件和目录范围；与读共用同一套文件级权限检查路径
命令行   ─── 独立维度，决定沙盒内/外执行
工具     ─── 独立维度，决定能否调用；内部读写复用读/写权限
收发消息 ─── 独立维度，可限制 Agent 或 User 收发消息
网络     ─── 独立维度，控制 outbound 网络访问
跨Agent  ─── 独立维度，控制 Agent 间通信
配置写入 ─── 独立维度，永远高危，只能单次审批
```

### 读

限制 Agent 可读取的文件和目录范围。通过文件操作表达，同时约束工具内部的读取行为。

### 写

限制 Agent 可写入的文件和目录范围。通过文件操作表达，同时约束工具内部的写入行为。读和写共用同一套文件级权限检查路径。

### 命令行

控制 Agent 可执行的命令白名单。有权限的命令运行在沙盒外（宿主环境），无权限的命令和所有脚本运行在沙盒内。命令白名单支持放行和阻止两种参数。

### 工具

控制 Agent 是否能调用某个工具。工具内部的读写操作复用读/写权限判定路径，不单独另算。

LLM 输出的普通文本消息不经过工具模块解析，直接走消息通道。只有匹配到工具调用格式的输出才进入工具模块。

### 收发消息

从权限维度看属工具类管控——可以限制 Agent 或 User 能否收发消息。但 IM 消息的格式解析和路由是 IM Processor 的职责，属不同层面的概念。

### 网络

控制 Agent 可访问的网络资源（主机、端口）。默认 Deny——所有网络访问需显式授权。

### 跨 Agent 通信

控制 Agent 之间是否可以通信（spawn、消息传递等）。默认 Deny。

### 配置写入

控制 Agent 能否写入系统配置文件。默认 Deny。此维度永远高危，只能走单次审批，不能被加入白名单。

## 数据流

```
Agent 工具调用 → tools 模块
  │
  ├─ 工具维度检查：Agent 是否有权调用此工具
  │   ├─ Deny → 终止
  │   └─ Allow → 继续
  │
  └─ 工具内部操作
        │
        ├─ 文件读写 → 读/写维度检查（复用同一路径）
        ├─ 命令执行 → 命令行维度检查
        ├─ 网络访问 → 网络维度检查
        ├─ Agent 间通信 → 跨 Agent 维度检查
        └─ 配置写入 → 配置写入维度检查
```

## 模块关系

- **上游**：tools 模块（在工具调用时触发各维度检查）
- **下游**：无（维度定义不调用其他模块，仅被权限引擎引用）
- **相关**：权限引擎（评估规则时以维度为单位匹配操作类型）
