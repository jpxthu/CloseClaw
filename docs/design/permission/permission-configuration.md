# 权限配置管理

## 概述

权限配置管理负责权限规则的修改、持久化、热加载和配置文件的访问保护。所有权限修改只能通过权限系统自身完成。

## 架构

### 修改入口

权限配置（规则文件加模板文件）的唯一修改入口是权限系统。配置文件不在 workspace 路径下，Agent 不可读不可写。

Owner 通过两种方式修改权限：

**斜杠指令**：Owner 在 IM 或 CLI 中发送斜杠指令，被 Gateway 硬拦截后在守护进程内直接执行，不进入 Agent Session。支持添加白名单、添加 Deny 规则、添加命令白名单等操作。每个指令在执行前验证发送者为 Owner。

**审批白名单**：Owner 在处理审批卡片时选择"加入白名单"，触发权限系统写入规则。高危操作无此选项。

### 热加载

权限变更立即生效。权限引擎支持热加载规则集——变更后下一次权限检查即使用新规则，Agent 无需重启。

### 配置保护

以下硬编码规则不可被任何配置覆写：

- 权限配置文件的写入 → Owner 独占
- 权限配置目录整体对 Agent → 禁止访问
- Owner 无法禁用自身身份校验
- 不存在"绕过权限系统"的选项

### 新建 User

非 Owner 用户创建需 Owner 审批。审批时 Owner 可选择该用户的初始权限集合（如授予收发消息权限）。创建后 User 可通过其绑定的 IM 渠道与 Agent 交互。

### Workspace 路径安全

Session 启动时自动为 Agent-User 组合创建 workspace 路径。路径中的 Agent ID 和 User ID 必须通过路径穿越校验：拒绝含 `../` 或绝对路径前缀等危险模式，防止用特殊字符绕过 workspace 隔离。并行启动时目录创建幂等，无竞态问题。

### 版本快照隔离

审批期间 Owner 修改权限规则不影响当前审批结果。审批请求入队时记录当前规则集版本号，审批响应时使用该版本的规则进行评估。

## 数据流

```
权限修改触发
  │
  ├─ Owner 斜杠指令
  │     │
  │     ▼
  │   Gateway 硬拦截 → 验证发送者 = Owner
  │     │
  │     ▼
  │   权限系统写入规则文件
  │
  └─ 审批白名单
        │
        ▼
     Owner 审批卡片选择"加入白名单"
        │
        ▼
     权限系统写入规则文件
        │
        ▼
     热加载：权限引擎 reload_rules()
        │
        ▼
     新规则立即生效
```

关键分支：
- 斜杠指令发送者非 Owner → 拒绝
- 写入的路径包含穿越字符 → 拒绝

## 模块关系

- **上游**：Gateway（斜杠指令拦截后传入）、审批工作流（白名单选项触发）
- **下游**：权限引擎（热加载通知）、文件系统（规则持久化）
- **相关**：审批工作流（白名单修改触发、版本快照隔离）
- **无关**：Agent Session（权限修改不进入 Session 上下文）
